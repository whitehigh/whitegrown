# 缺陷分析 Agent

## 项目目标

辅助 QA 对 Bug 描述、复现步骤、日志和业务模块进行整理，快速形成可提交给研发的缺陷分析信息。

## 输入

- Bug 标题和描述
- 复现步骤
- 日志片段
- 影响平台、地区或版本
- 当前期望结果和实际结果

## 输出

- 缺陷分类
- 可能影响范围
- 建议排查方向
- 需要补充的信息
- JIRA 字段草稿

## Agent 工作流

1. 提取关键信息
2. 判断模块和缺陷类型
3. 检查复现步骤是否完整
4. 结合日志或数据线索给出排查建议
5. 生成标准化缺陷描述
6. 由 QA 人工确认后提交

## 适用场景

- 高频 Bug 归类
- 缺陷描述标准化
- 新版本回归问题整理
- 海外版本多地区问题归纳
